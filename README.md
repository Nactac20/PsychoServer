# PsychoServer - Сервис по работе с психологами

## Описание проекта

Серверная часть сервиса для записи к психологам. Реализована бизнес-логика с многопоточной обработкой данных, сетевым взаимодействием по протоколу TCP и хранением в SQLite.

Проект выполнен в рамках дисциплины "Многопоточное программирование". 

---

# Функциональность

## Этап 1: Разработка бизнес-логики сервера

### Пользователи
- Регистрация клиентов и психологов
- Поиск пользователей по email и ID
- Валидация паролей
- Защита от дублирования email

### Расписание
- Добавление слотов для записи
- Просмотр свободных слотов (только будущие, не забронированные)
- Автоматическое исключение просроченных слотов

### Бронирование
- Потокобезопасное бронирование слотов
- Защита от двойной записи (транзакции + мьютексы)
- Создание сеанса после успешного бронирования
- Проверка существования клиента и психолога
- Запрет бронирования просроченных слотов

### Логирование
- Многопоточный логгер с записью в файл и консоль
- Уровни: INFO, WARNING, ERROR
- Временные метки для каждого сообщения

---

## Этап 2: Сетевое взаимодействие (Network Edition)

### Сервер и протокол
- Многопоточный TCP-сервер на базе **Boost.Asio**.
- Обработка запросов в пуле потоков (**Thread Pool**).
- Обмен данными в формате **JSON** (**Boost.JSON**).
- Поддержка множественных одновременных подключений.

### Чат и уведомления
- Пересылка сообщений между клиентом и психологом в реальном времени.
- Система получения уведомлений о новых записях и событиях.

## Сборка и запуск

### Требования
- CMake 3.16+
- Компилятор с поддержкой C++17 (MSVC, GCC, Clang)
- SQLite3 (автоматически подключается)
- Boost 1.75+

### Сборка
```powershell
mkdir build
cd build
cmake ..
cmake --build . --config Release
```

### Запуск
```powershell
# Запуск сервера:
./Release/psycho_server.exe

# Запуск тестов:
ctest -C Release
```

## Многопоточность и безопасность
- **Thread Pool**: Параллельная обработка клиентских запросов.
- **Мьютексы (std::recursive_mutex)**: Защита общих данных и ресурсов.
- **Транзакции SQLite (BEGIN IMMEDIATE)**: Гарантия атомарности операций.
- **Проверка sqlite3_changes()**: Контроль обновлений данных.
- **Тест с 10+ потоками**: Доказательство корректности при высокой нагрузке.
